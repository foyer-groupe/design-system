Atomic building principle

Foyer Design System (FDS) is built with the same principle than organism. Small molecules can create beautiful living beings.
Each User Interface (UI) element can be used standalone or can be associated to other to build more complex component, called an organism.
Each UI element is encapsulate in a NPM library to be easily imported in your project and to manage his dependencies.

FAQ
  • When do we need to ask the FDS team to add the stuff we need ?

    Well you don't. All the designers have the FDS components in their design tools, and thus when they create a new mock, they know already if something is missing. They will tell the FDS team what is needed, and when the devs are supposed to have it (basically at the start of a sprint). As making mistakes is human nature, if it ever happens that one feature is missing on a component, you should discuss it with the FDS team directly.


  • How to use the components' Javascript ?

    There is NO JavaScript shipped with the packages. At the moment, components who rely heavily on JS aren't in the FDS (i.e. carousel, ...). Sometimes, you can find a JS tab in the documentation (open/close modal, ...), it is mainly used to make the doc working, and should be redone completly inside your JS files. We have been working with MEO to style some components they have been doing. You can find a list here.
    We also have started a web component library based on StencilJs.


  • When should I use custom CSS ?

    Basically, some details (i.e. position of an element inside a Panel, ...) may not be achievable directly with the FDS components. You should always have a look first at the utilities package which offers several classes acting as inline style.
    If you have a specific request, you can see with the FDS team if they have some time to help out. It happened that the team did some work which won't be reused through several projects. That work lies in the Labs. When you do integrate what's in the Labs, you need to copy the source from the Github and use it directly inside your project.


  • How do I report bugs ?

    If this is a critical issue, you should always come to see the team directly. Otherwise, the best way is to create a case on Github.
